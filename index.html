Enhanced Recommender System for Cold Start Mitigation

Overview:

Developed an enhanced Neural Collaborative Filtering (NCF) framework to tackle user and item cold start problems in recommender systems. Integrated user demographics and item metadata with self-attention and gating mechanisms to dynamically filter, weight, and incorporate side information. The framework outperformed traditional methods like Matrix Factorization (MF), SVD, and baseline NCF, delivering accurate recommendations even in sparse-data environments.

Dataset:

  • MovieLens 1M dataset: 1M ratings across 6,000+ movies from 6,000+ users.
  • User features: age, gender, occupation.
  • Item features: movie genre, release year.
  • Preprocessing: rating rescaling, metadata embedding, and stratified splits to simulate sparse-data scenarios.

Objectives:

  • Mitigate cold start for users, items, and joint user–item cases.
  • Leverage side information embeddings for richer user/item representations.
  • Apply self-attention to prioritize relevant features dynamically.
  • Use gating to selectively incorporate metadata and reduce noise.
  • Validate improvements against baselines and ablation studies.

Technical Approach:

  • Model Design: Enhanced NCF with metadata embeddings, self-attention, and gating mechanisms for robust recommendations.
  • Training & Optimization: Python with PyTorch/TensorFlow; Adam optimizer; Bayesian optimization for hyperparameter tuning.
  • Evaluation: MAE and RMSE metrics; ablation studies confirmed importance of attention and gating.

Key Insights:

  • Up to 7.5% reduction in MAE and 5.6% reduction in RMSE compared to baseline models.
  • Self-attention effectively prioritized informative features; gating reduced noise from irrelevant metadata.
  • Side information + attention + gating combination enabled robust recommendations in sparse-data environments.

Tools Used:

  • Python, PyTorch, TensorFlow, NumPy, Pandas, Scikit-learn
  • Bayesian Optimization: Optuna/Hyperopt
  • Visualization: Matplotlib, Seaborn
  • Methodology: CRISP-DM workflow

Checkout Optimizer: A Full-Stack A/B & Bandit Experimentation Framework

Overview:

Developed a full-stack experimentation framework to evaluate checkout design changes using advanced statistical methods. This project went beyond traditional A/B testing by integrating CUPED variance reduction, Bayesian inference, sequential monitoring, uplift modeling, and multi-armed bandit simulations. The framework enabled rapid, data-driven decision-making on whether replacing the “Continue” button with a sticky progress bar improved Gross Profit per Visitor (GPV).

Key Objectives:

  • Measure the causal impact of a sticky progress bar on Gross Profit per Visitor (GPV).
  • Reduce experiment variance and increase sensitivity with CUPED adjustments.
  • Enable early stopping through sequential monitoring while preserving statistical validity.
  • Identify heterogeneous treatment effects across user segments via uplift modeling.
  • Benchmark classical A/B testing against adaptive multi-armed bandit strategies.

Technical Approach:

  • Data Simulation: Built a synthetic data generator replicating real-world checkout flows, user heterogeneity, and operational guardrails.
  • Frequentist & Bayesian Analysis: Applied CUPED for variance reduction, p-value testing for significance, and Bayesian posterior inference for probabilistic insights.
  • Sequential Monitoring: Implemented O’Brien–Fleming boundaries to detect early stopping opportunities.
  • Causal Uplift Modeling: Estimated Conditional Average Treatment Effects (CATE) to uncover high-impact user segments (e.g., mobile users, India-based users).
  • Bandit Algorithms: Compared Thompson Sampling, UCB1, and Epsilon-Greedy with fixed A/B testing to optimize traffic allocation.

Key Insights:

  • CUPED-adjusted analysis revealed a 7.55% lift in GPV, statistically significant with p < 0.001.
  • Bayesian inference showed a 99.96% probability that treatment outperformed control.
  • Sequential analysis allowed early stopping by Day 8, reducing experiment time by nearly 40%.
  • Uplift modeling revealed strong heterogeneity — mobile and Indian users showed the largest positive effects.
  • Thompson Sampling outperformed other policies in bandit simulations, achieving 100% PCS and highest cumulative rewards.

Tools Used:

  • Python: pandas, numpy, statsmodels, scikit-learn, scipy for experiment design and analysis.
  • Jupyter Notebooks: Interactive exploration, causal modeling, and reporting.
  • SQL: Schema design and quality checks for experiment pipelines.
  • Streamlit: Interactive dashboard for experiment visualization and stakeholder communication.
